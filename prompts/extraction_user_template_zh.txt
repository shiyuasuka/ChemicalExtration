请执行以下提取工作流，并严格遵守系统规则。

以实验室提取语义作为真值来源。
- `Paper_Metadata` 和 `items` 定义了目标语义结构。
- 运行时 `g4_bindings` 输出仅是这些实验室语义的运行时兼容编码。
- 每个运行时 `g4_bindings` 条目必须与用户数据集中的最终扁平 `records[]`
  条目 1:1 对应。

第一步：全局元数据与角色区分

- 将论文标题和 DOI 提取到根节点 `Paper_Metadata` 对象。
- 将本文自身的测量结果提取为 `Target`。
- 若存在明确引用的基准值，可将其提取为独立的 `Reference` 条目，
  但绝不能将其混入 `Target` 条目。

第二步：条目分解

- 一个条目 = 配体 + 序列 + 方法 + 活性的唯一组合。
- 当配体身份、序列、方法、活性类型、缓冲液、浓度或温度发生变化时，拆分为新条目。
- 若表格某行 / 条件含多个端点列，为每个非空端点单元格各出一条条目。
- 在 `value` 字段中保留端点标签
  （如 `Delta Tm2=6.4 ℃`，而非仅 `6.4 ℃`）。
- 将行标签 / 测定条件（如 `D/N=0.5`、基线游离 DNA Tm、配对结合态 Tm 或
  温度注释）放入 `comments`。
- 若同一配体–序列对在多种条件下被测量，为每个不同测量环境各出一条条目。
- 若某变体被明确命名但证据稀疏，仍保留稀疏条目。
- 不要为原始支撑性观察量创建独立条目，除非论文将其明确呈现为最终数据集级端点。
- 当基准测试将计算型、分子水平和细胞水平的端点视为最终记录时，为其创建独立条目。

第三步：配体提取（`Ligand_Info`）

- 填写 `ligand_id`、`ligand_name`、`ligand_name_std` 和 `ligand_synonyms`。
- `ligand_id` 须遵循 G4Lxxxx 模式。
- 永不编造占位符 ID（如 `G4L0001`）。
- 若无可信的整理映射，将 `ligand_id` 置为 null。
- 当论文将多个名称 / 标识符一起打印时，`ligand_name` 保留完整报告别名字符串，
  不要缩减为短昵称。
- 论文名称字段中的多个别名应以分号分隔；将其拆分到同义词数组中。
- 只要有证据，就提取 `counter_ion` 和 `metal_ion`。

第四步：序列提取（`Sequence_Info`）

- 填写 `sequence`、`sequence_name` 和 `sequence_type`。
- `sequence` 须为确切的核苷酸字符串（DNA: A/T/G/C；RNA: A/U/G/C），
  仅当可明确恢复时填写；否则置为 null。
- 绝不将别名（如 "ZGQ1"、"HGQ-NV-L"）放入 `sequence` 字段。
  别名仅属于 `sequence_name`。若无核苷酸字符串，将 `sequence` 置为 null。
- 对于 RNA 序列，使用 U（尿嘧啶），不用 T。若论文或上下文将序列标识为 RNA
  （如 "RNA G-quadruplex"、"viral RNA"、"mRNA region"），则在 `sequence`
  字段中使用 A/U/G/C 表示法。
- 当论文明确给出重复计数时，将重复缩写展开为显式序列
  （如 `[d-(TTAGGGT)]4` → `TTAGGGTTTAGGGTTTAGGGTTTAGGGT`）。
- 当别名和精确序列同时存在时，两者均保留：
  别名 → `sequence_name`，精确字符串 → `sequence`。
- 保留修饰标记（如 5'-fluoro、2'-O-methyl），作为序列字符串的一部分或放入 comments。
- 保留复合序列标签，如 `Pu24; PDB ID: 2MGN`（论文以此方式报告时）。
- `sequence_type` 使用数据集规范值；本基准中，优先使用 `DNA`、`RNA` 或 null。

第五步：活性提取（`Activity_Info`）

- 填写 `activity1`、`activity2` 和 `value`。
- `activity1` 须使用数据集顶级标签：
  `Interaction`、`Activity at Molecular Level` 或 `Activity at Cellular Level`。
- 对序列级生物物理测定，优先使用 `activity1 = Interaction`。
- 使用基准测试的稳定方法到标签映射：
  - TRAP → `Activity at Molecular Level` + `Enzyme`
  - ChIP / IF / FISH → `Activity at Molecular Level` + `DNA Damage`
  - 受体抑制 / hERG 面板 → `Activity at Molecular Level` + `Gene Expression`
  - SRB / 细胞生长抑制 → `Activity at Cellular Level` + `Cytotoxicity`
- `activity2` 尽可能使用数据集端点标签：
  `Binding`、`Stabilization`、`Recognition`、`Enzyme`、`Gene Expression`、
  `DNA Damage` 或 `Cytotoxicity`。
- 对 ITC / SPR / NMR 结合结论 / 对接 / MM-PBSA / MSM / MD 输出，
  优先使用 `Binding`，除非论文明确将端点框定为 `Recognition` 或 `Stabilization`。
- 对荧光选择性陈述（如 `>100 fold higher than duplex control`、
  "preferentially binds G4 over duplex"）或类似定性 G4 选择性发现（无拟合 KD/Ka），
  使用 `Recognition`。不得对这些定性选择性陈述使用 `Binding`。
  关键：若对多个命名 G4 序列报告了选择性，则为每个序列名称各出一条 Recognition 记录。
- `value` 须原样保留原始报告字符串，包含科学计数法、不等式、范围和单位
  （如 "Kd = 2.3 µM"、"ΔTm = 12 °C"、"> 50 µM"、"1.2×10⁻⁸ M"）。
- 只要论文已打印特定标签，就在原始 value 中保留端点标签，如
  `Delta Tm2=6.4 ℃`、`Ka1=1E8 M−1`、`Docking Score=-9.236 kcal/mol`、
  `Rise=3.43±0.07` 或 `Binding mode: intercalation`。
- 关键 — MD / 计算型记录粒度：当计算结果块对同一配体–序列组合报告多个不同指标时
  （如 Rise、H-twist、对接评分、结合模式占比、每种结合模式的 MM-PBSA ΔE），
  为每个不同指标各出一条独立记录，并在 `value` 中保留指标标签。
  不得将多个指标合并为一个 value 字符串。

第六步：实验条件（`Experimental_Conditions`）

- 尽可能使用数据集规范名称提取 `method`：
  `Differential Scanning Calorimetry (DSC)`、`Nuclear Magnetic Resonance (NMR)`、
  `Molecule Dynamics (MD) Simulation`、`Surface Plasmon Resonance (SPR)`、
  `Isothermal Titration Calorimetry (ITC)`、`Circular Dichroism (CD)`、
  `Fluorescence Spectroscopy (Fl)`、`Molecular Docking Calculation`、
  `Markov State Model (MSM) analysis`、`UV-Vis Absorption Spectroscopy`、
  `Molecular Mechanics Poisson Boltzmann Surface Area (MM-PBSA)`、
  `Telomeric Repeat Amplification Protocol (TRAP) Assay`、
  `Sulforhodamine B assay (SRB)`、
  `Chromatin Immunoprecipitation (ChIP)`、
  `Immunofluorescence (IF)`、
  `Fluorescent in Situ Hybridization (FISH)`、
  `Receptor Inhibition Study`。
- 保留与结果块关联的基准方法。除非论文明确将其呈现为独立测定，否则不要将一个
  计算结果块人为拆分为额外的方法。
- 只要有报告，将 `buffer` 提取为完整组成字符串。
- 带单位提取 `sample_concentration`。
- 有报告时提取 `instrument` 型号。
- 在 `comments` 中记录其他注记。

第七步：表格对齐与泛化检查

- 通过表头锁定、行锁定、温度锁定和单位锁定重建数值。
- 不得从模糊 OCR 或未标注的图线中猜测数值。
- 不得将引用的前人工作值转为当前论文的 Target 记录。
  "earlier"、"previously reported"、"has been obtained" 加引文是该值
  不属于 Target 记录的强烈信号。
- 不得纳入仅作比较用途的参考 / 对照配体（如已知基准化合物），
  除非论文明确将其呈现为与主要配体系列同等的实验结果。
- 不得将低级诊断 / 支撑性观察量转为独立记录，除非论文明确将其呈现为最终端点。
- 不得提取合成中间体、化合物表征行、熔点、常规 NMR 归属段落或其他非测定型
  化学元数据。
- 不得提取双链 / 仅对照行，除非论文明确将其视为基准级主要记录。
- 不得提取电生理 / 膜片钳结果。膜片钳测定是离子通道端点，不是 G4 配体–核酸
  结合 / 稳定化端点。method 为 "patch clamp"、"voltage clamp" 或类似电生理
  方法的任何记录须静默忽略。
- 不得提取 MM-PBSA 能量子项。仅提取最终总结合能（ΔG_bind、Binding energy total）。
  ΔE_SUR、ΔE_PBELE、ΔE_PBTOT、ΔEvdW、ΔEele 等能量分解项不是数据集端点，
  必须忽略。
- 绝不生成 `value` 字段仅包含补充数据指针或表格引用的记录（如 "detailed
  values listed in Table S7"、"see supplementary"、"higher affinity for GQ
  sequences; detailed values listed in Table S7"）。这些字符串不是测量值，
  必须丢弃。仅当实际结果出现在已提取文本中时，才可输出记录。
- 绝不生成 value 字段仅包含定性亲和力比较字符串（如"对 GQ 序列亲和力更高"、
  "高于双链对照亲和力"）而无拟合 Kd/Ka/IC50 的 Binding 记录。
  Binding 记录必须有定量常数。若论文仅给出定性 G4 选择性陈述而无数值，
  应改为 Recognition 记录（如有命名 G4 序列）或直接忽略。
- 在最终输出前，构建全部明确配体名称、序列标签和测量条件的完整列表。
- 确保每个有明确支撑的测量都有对应条目。
- 不要过拟合于某一种表述模式；在不同论文排版下保持通用的提取行为。

第八步：强制最终自审

- 检查 Target / Reference 纯洁性。
- 检查未明确遗漏任何配体、序列或测量列。
- 检查测量类型未与活性标签错配
  （如 Kd 不应在无证据的情况下出现在 Inhibition 下）。
- 检查未编造占位符配体 ID。
- 检查表格多列端点已被正确拆分。
- 检查最终载荷为合法 JSON。
- 覆盖度：若对接 / MD 表含 N 个配体行，确认已生成 N 条 Binding 记录
  （每个配体每个对接指标各一条）。不得提前终止。
- 覆盖度：若对接表格同时含有 "Docking Score change" 列，确认每个有数据的
  配体行已额外生成一条 Docking Score change 记录。
- 覆盖度：若细胞毒性表含 M 个细胞系列，确认每个配体行已生成 M 条
  Cytotoxicity 记录。
- 覆盖度：若荧光选择性实验测试了 K 个命名 G4 序列，
  确认已生成 K 条 Recognition 记录。
- 覆盖度：若 DSC 或 CD 表含 Tm2/Tm3/Tm4/Tm5（或 Delta Tm2/3/4/5）列，
  且有多个 D/N 条件行，确认每个 D/N 行 × 每个非空列已各自生成 Stabilization 记录。
- 覆盖度：若 ITC 或其他实验行报告了 Ka1 和 Ka2（或 Kb1/Kb2、Kd1/Kd2），
  确认该配体–序列对已生成两条独立 Binding 记录。
- 排除检查：确认输出中不含任何膜片钳 / 电生理记录。
- 排除检查：确认不含任何 MM-PBSA 记录其 value 仅包含能量子项
  （ΔE_SUR、ΔE_PBELE、ΔE_PBTOT、ΔEvdW）而非总结合能。
- 质量检查：扫描所有已生成记录，拒绝任何 `value` 字段仅包含数据位置引用
  而非实际报告测量值的记录。
- 质量检查：扫描所有 Binding 记录——任何仅含定性亲和力陈述（无数值 Kd/Ka/IC50）
  的记录须重新分类为 Recognition 或删除。
- 质量检查：扫描所有 method = `Mass spectrometry` 的记录——若其 value
  包含 "first passage time"、"pathway flux"、"interstate flux" 或 "MSM"，
  须更正 method 为 `Markov State Model (MSM) analysis`。

重要输出规则：

- 遵循以 `Paper_Metadata` 和 `items` 为根的实验室模式语义，
  但输出以 `g4_bindings` 为根的运行时内部模式。
- 使用与系统提示词一致的字段映射，以保证下游流水线阶段的兼容性。
- 不得输出任何散文或 Markdown 格式内容。

示例 F（标签驱动的拆分是强制要求）
输入（抽象化）：
- 同一配体 G4L0065，对应序列：c-MYC、hTel、KRAS。
决策：
- 输出 3 条独立条目（每个序列标签各一条），而非合并为一条。
错误做法：

示例 G（条件级拆分是强制要求）
输入（抽象化）：
- 同一配体–序列分别用 ITC（Kd）和 CD 熔解（ΔTm）测量。
决策：
- 输出 2 条独立条目，每种方法 / 测量类型各一条。
错误做法：
- 一条条目中同时包含 Kd 和 ΔTm 值。

示例 H（缓冲液条件拆分）
输入（抽象化）：
- 同一配体–序列分别在 Tris-HCl 和 PBS 缓冲液中测量。
决策：
- 输出 2 条独立条目，每种缓冲液条件各一条。
错误做法：
- 一条条目，合并或平均两种缓冲液条件。

示例 I（一篇论文中含多个配体）
输入（抽象化）：
- 论文报告了 G4L0065 和 G4L0140 针对同一序列的结合数据。
决策：
- 输出 2 条独立条目，每个配体 ID 各一条。
错误做法：
- 一条合并条目，包含两个配体。

示例 J（浓度系列）
输入（抽象化）：
- 配体 G4L0065 针对 c-MYC，在 6 种不同配体浓度下报告 IC50。
决策：
- 输出 1 条条目，包含 IC50 值；额外的浓度细节放入 comments。
错误做法：
- 输出 6 条独立条目（浓度不是条目拆分维度，除非论文将每个浓度视为

示例 K（不得混合测量类型）
输入（抽象化）：
- 同一篇论文报告了 G4L0065–c-MYC 的 Kd（ITC）和 IC50（细胞活力）。
决策：
- 输出 2 条条目：一条生物物理结合（Kd），一条细胞活性（IC50）。
  活性标签不同（Binding vs Cytotoxicity）。
错误做法：
- 一条条目，在同一活性标签下混合 Kd 和 IC50。

示例 L（保留稀疏条目）
输入（抽象化）：
- 论文提到 G4L0200–KRAS 的相互作用，但仅报告了 ΔTm 值，无序列细节或缓冲液。
决策：
- 输出 1 条稀疏条目，填写可用字段；不得编造缺失数据。
错误做法：
- 因记录稀疏而跳过该记录。

示例 M（表格列拆分是强制要求）
输入（抽象化）：
- 表格行 `D/N=0.5` 包含 `Delta Tm2=6.4`、`Delta Tm3=10.9`、`Delta Tm5=24.3`。
决策：
- 输出 3 条独立条目，每个非空端点列各一条。
- 将 `D/N=0.5` 及配套的基线 / 结合态 Tm 详情放入 `comments`。
错误做法：
- 一条合并条目，`value` 中包含 `6.4, 10.9, 24.3`。

示例 N（不得将支撑性观察量提取为独立端点）
输入（抽象化）：
- 论文仅将化学位移变化、NOE 计数、扩散系数和荧光猝灭作为结合模型的支撑证据报告。
决策：
- 不要为这些观察量各出一条记录，除非论文明确将其呈现为最终数据集端点。
- 若直接陈述了结合化学计量、结合位点或结合模式，则应生成相应记录。
错误做法：
- 分别为化学位移扰动、NOE 计数、扩散系数、荧光猝灭波长和 CD 谱带变化
  各出一条记录。

示例 O（不得将引用的前人工作转为 Target）
输入（抽象化）：
- 讨论部分写道："Using the method of Job Plot, the highest binding stoichiometry
  has earlier been obtained as 2:1 in adriamycin-G4 complex (Raje et al., 2019a)."
决策：
- 将此视为前人工作背景，而非当前论文的 Target 值。
错误做法：
- 将结合亲和力或化学计量值作为本文测量结果输出。

示例 P（计算结果可以是最终记录）
输入（抽象化）：
- 论文报告了 MM-PBSA、MSM 和 MD 输出，如 `Rise=3.43±0.07`、
  `Interstate fluxes=1:1`、`H-twist=24.18±2.60` 和 `Binding mode: intercalation`。
决策：
- 使用拥有这些输出的基准方法名称，各自发出独立的 `Binding` 记录。
错误做法：
- 因为是计算型结果而丢弃，或无证据地重新标注为 `Stabilization`。

示例 Q（细胞 / 分子水平记录在范围之内）
输入（抽象化）：
- 论文报告了 TRAP `IC50=9 μM`、SRB `IC50=4.85 μM`、ChIP 端粒损伤
  和 FISH 端粒异常结果。
决策：
- 将其分别作为 `Enzyme`、`Cytotoxicity` 和 `DNA Damage` 记录输出；
  当无精确核酸序列报告时，sequence 置为 null。
错误做法：
- 因不是直接的配体–序列生物物理测定而丢弃。

示例 R（荧光实验中 Recognition 与 Binding 的区分）
输入（抽象化）：
- 荧光实验得出结论：GQ RNA 的亲和力相对于双链对照 `>100 fold higher`，
  但未给出拟合 KD/Ka。
决策：
- 使用 `activity2 = Recognition`。
错误做法：
- 编造数值 KD 或强行将记录归入通用 `Binding`。

示例 S（排除合成 / 表征内容）
输入（抽象化）：
- 合成部分报告了中间体化合物熔点、1H NMR 峰和 HRMS 确认。
决策：
- 不得生成数据集记录。
错误做法：
- 将这些合成表征行输出为相互作用记录。

示例 T（SRB 每细胞系拆分是强制要求）
输入（抽象化）：
- SRB 表格含列：配体 | IC50 (HeLa) | IC50 (A549) | IC50 (SKOV-3)，
  适用于配体 G4L0072、G4L0088、G4L0089。
决策：
- 输出 9 条独立的 Cytotoxicity 记录（3 个配体 × 3 个细胞系）。
- 在 value 中保留细胞系标签，如 `IC50 (HeLa) = 4.2 µM`。
错误做法：
- 每个配体一条合并记录，将三个 IC50 值合并在一起。
- 因数值相近而跳过某个细胞系列。

示例 U（MD / 对接表：必须穷举所有配体行）
输入（抽象化）：
- 分子对接表列出 40 个化合物（编号 1–42，跳过 2 个），
  每个化合物针对同一序列 Pu24 各有一个对接评分。
决策：
- 输出 40 条独立的 Binding 记录，每个化合物行各一条，
  `method = Molecule Dynamics (MD) Simulation` 或 `Molecular Docking Calculation`
  （视情况而定），`value = "Docking Score=<X> kcal/mol"`。
错误做法：
- 仅输出评分最高的 3–5 条记录。
- 将所有评分合并为一条记录。

示例 V（MSM ≠ 质谱）
输入（抽象化）：
- 论文报告了马尔可夫状态模型分析，包含平均首次通过时间：
  "unbound → top binding = 1.6±0.2 μs"、"unbound → side = 4.3 μs"。
决策：
- 使用 `method = Markov State Model (MSM) analysis`，activity2 = Binding。
- 为每个不同的通路 / 通量指标各出一条记录。
错误做法：
- 将 MSM 态间通量值的 method 标注为 `Mass spectrometry`。

示例 W（RNA ITC：使用 U 而非 T；Ka1 和 Ka2 各出一条记录）
输入（抽象化）：
- ITC 实验针对 RNA G4 序列 HGQ-NV-L（序列：UUUAAGGAGAACGGG…），
  同一表格行报告了 Ka1 = 3.52E6 M−1 和 Ka2 = 4.58E4 M−1。
决策：
- 为同一配体–序列对输出 2 条 Binding 记录：
  记录 1：value = "Ka1=3.52E6 M−1"，method = ITC，sequence 使用 U 字符。
  记录 2：value = "Ka2=4.58E4 M−1"，method = ITC。
- 在 sequence 字段中使用 A/U/G/C（RNA）。
错误做法：
- 将两个 Ka 值合并为一条记录。
- RNA 序列字符串中使用 T 而非 U。

示例 X（Recognition：每个命名 G4 序列各出一条记录）
输入（抽象化）：
- 荧光选择性论文测试了配体针对 G4 序列 ZGQ1、ZGQ2、ZNS3、ZNS5B 的结果，
  每个序列均报告了 ">100 fold higher affinity than duplex control"，
  且给出了 RNA 核苷酸序列。
决策：
- 输出 4 条独立的 Recognition 记录（每个 G4 序列标签各一条）。
- 使用 activity2 = Recognition；将 sequence 设为 RNA 核苷酸字符串。
错误做法：
- 将四个序列合并为一条记录。
- 因发现是定性选择性而非拟合 KD，错误地使用 activity2 = Binding。

示例 Y（不得生成摘要型 / 表格指针型 value 记录）
输入（抽象化）：
- 文本写道："TMPyP4 showed higher affinity for GQ sequences; detailed values
  listed in Table S7"，涉及序列 ZNS3、ZNS5A、ZNS5B。
决策：
- 不得为这三个序列标签生成记录，因为已提取文本中没有实际测量值。
  value 字段只会包含表格引用，而非结果。
错误做法：
- 输出 value = "Higher affinity for GQ sequences; detailed values listed in
  Table S7" 的记录。这是指针字符串，不是测量值。

示例 Z（双链 SPR 是对照记录——不得作为独立 Binding 记录输出）
输入（抽象化）：
- SPR 表格显示：h-Tel G4 → K = 0.83×10^7 M^-1（主要结果）；
  双链 DNA 对照 → K = 0.06×10^7 M^-1（仅作选择性比较）。
决策：
- 仅为 h-Tel G4 输出 1 条 Binding 记录。
- 不得为双链 DNA 对照行单独输出 Binding 记录。
  该双链行仅用于记录选择性背景。
错误做法：
- 输出 2 条 Binding 记录：一条针对 h-Tel，一条针对双链 DNA。

示例 AA（有拟合 Kd 时，Binding 优先于 Recognition）
输入（抽象化）：
- 荧光竞争测定对 BRACO-19 vs ZGQ1 报告了：
  (1) 定性选择性陈述：">100 fold higher affinity than duplex control"；
  (2) 同一荧光实验中的拟合结合常数：Kd = 0.09 µM。
决策：
- 输出 1 条 Binding 记录，value = "Kd=0.09 µM"（不用 Recognition）。
- 拟合 Kd 永远优先于定性选择性陈述，即使方法是荧光光谱也如此。
  当同一序列同时有选择性陈述和拟合 Kd 时，使用 Binding 并填写 Kd 值。
- 按序列逐个判断：ZGQ1 有 Kd=0.09 µM → Binding；ZGQ2 有 Kd=0.708 µM → Binding；
  ZGQ4 有 Kd=0.035 µM → Binding。只有完全无拟合 Kd 的序列才用 Recognition。
错误做法：
- 在 Kd 已有报告的情况下，输出 Recognition 记录并填写选择性陈述作为 value。
- 对同一实验中的同一序列同时输出 Binding 和 Recognition 两条记录。

示例 AB（MSM 方法——永远不得与质谱混淆）
输入（抽象化）：
- 论文报告马尔可夫状态模型分析，包含：
  "mean first passage time (unbound → top binding) = 1.6±0.2 μs"、
  "pathway flux: 60% direct (unbound→top), 27% via side, 13% via bottom"。
决策：
- 输出 3 条 Binding 记录，每条均使用 method = "Markov State Model (MSM) analysis"。
- 记录 1：value = "Mean First Passage Time (unbound → Top direct) = 1.6 ± 0.2 µs"
- 记录 2：value = "Pathway flux: 60% (direct)"
- 记录 3：value = "Pathway flux: 27% (unbound → Side → Top)"
错误做法：
- 对上述任何记录使用 method = "Mass spectrometry"。
  MSM ≠ 质谱。"mean first passage time" / "pathway flux" 永远表示 MSM。

示例 AC（SRB：提取所有具有明确 IC50 值的化合物）
输入（抽象化）：
- SRB 表格列出化合物 6a、6b、7a、8a、8b、9a、10a…（均为本论文研究的新
  氟酮衍生物），表格含 A2780、SKOV-3、CH1 三个细胞系列。
- 某些化合物不活跃（IC50 > 20 µM），但仍给出了明确数值。
决策：
- 为每个具有明确 IC50 值的化合物（含不活跃化合物）×每个细胞系分别输出
  一条 Cytotoxicity 记录，value 中保留细胞系标签（如 "IC50 (A2780) = 4.85 µM"）。
- 不得跳过不活跃化合物（IC50 > 20 µM 同样是有效记录）。
- 仅当论文明确标注某行为 "positive control" 或 "reference drug" 时才跳过。
错误做法：
- 跳过 IC50 > 20 µM 的化合物。
- 每个化合物只出 1 条合并记录（将多个细胞系 IC50 合并在一起）。

示例 AD（DSC 或 CD D/N 条件拆分是强制要求）
输入（抽象化）：
- DSC（或 CD）表格含行：D/N=0.5、D/N=2.0、D/N=3.0；列：Delta Tm2、Delta Tm3、Delta Tm4、Delta Tm5。
- D/N=0.5 行：Delta Tm2=6.4、Delta Tm3=10.9、Delta Tm4=18.5、Delta Tm5=24.3。
- D/N=2.0 行：Delta Tm2=11.7、Delta Tm3=12.7、Delta Tm4=20.1、Delta Tm5=30.2。
- D/N=3.0 行：Delta Tm2=14.0、Delta Tm3=15.2、Delta Tm4=22.3、Delta Tm5=35.1。
决策：
- 输出 12 条独立的 Stabilization 记录（3 个 D/N 行 × 4 个非空列）。
- 每条记录：value = "Delta Tm2=6.4 ℃"（保留端点标签）；
  D/N 比值和配套基线 Tm 信息放入 `comments`。
- 此规则同时适用于 DSC 和 CD 熔解实验的 D/N 条件表格。
错误做法：
- 仅输出 4 条记录（每列一条，忽略 D/N 维度）。
- 仅输出 3 条记录（每行一条，将所有列合并）。
- 输出 1 条合并记录。
- 仅对 DSC 应用此规则，忽略 CD 表格的 D/N 条件。

示例 AE（对接评分变化是独立端点）
输入（抽象化）：
- 对接表格列出化合物 1–42（含 40 个有效行），均对接至同一序列 Pu24。
- 表格列：化合物编号 | 对接评分（kcal/mol）| 对接评分变化 vs DBD1（kcal/mol）。
- 参考化合物 DBD1：对接评分 = -8.891，无变化列（或变化 = 0）。
- 化合物 23 行：对接评分 = -9.146，对接评分变化 = -0.275。
决策：
- DBD1：输出 1 条记录，value = "Docking Score=-8.891 kcal/mol"。
- 化合物 23：输出 2 条独立记录：
  记录 1：value = "Docking Score=-9.146 kcal/mol"
  记录 2：value = "Docking Score change=-0.275 kcal/mol"
- 对所有同时具有两列数据的化合物重复上述逻辑。
错误做法：
- 将 "对接评分" 和 "对接评分变化" 合并为一条记录。
- 完全忽略 "对接评分变化" 列。
- 将对接评分变化放入 comments 而非作为独立的 value 输出。

示例 AF（MM-PBSA：仅输出总结合能，不输出能量子项）
输入（抽象化）：
- 论文报告配体 DBD1 与 Pu24 的 MM-PBSA 结果：
  ΔE_SUR (Top binding) = 20.3±0.3 kcal/mol
  ΔE_PBELE (Top binding) = 5.9±0.4 kcal/mol
  ΔE_PBTOT (Top binding) = -25.9±0.4 kcal/mol
  Binding energy (Top binding) = -16.9±2.6 kcal mol-1
决策：
- 仅输出 1 条 MM-PBSA Binding 记录，value = "Binding energy (Top binding) = -16.9±2.6 kcal mol-1"。
- 不得为 ΔE_SUR、ΔE_PBELE、ΔE_PBTOT 各输出独立记录——这些是能量分解子项，
  不是最终数据集端点。
错误做法：
- 输出 4 条独立 Binding 记录（每个能量项一条）。
- 保留 ΔE_SUR、ΔE_PBELE、ΔE_PBTOT 作为独立记录。

示例 AG（膜片钳 / 电生理必须排除）
输入（抽象化）：
- 论文同时报告了 G4 结合数据（ITC、CD）和通过膜片钳测定（patch clamp）对
  hERG 离子通道的影响。
决策：
- 正常提取 ITC 和 CD 记录。
- 不得为膜片钳结果输出任何记录——它们是离子通道端点，不是 G4 配体–核酸结合端点。
错误做法：
- 以 Gene Expression 或任何其他记录类型输出膜片钳数据。
- 将 "Patch Clamp Assay" 作为 method 出现在任何记录中。

示例 AH（ITC 无拟合 Kd 时：使用 Recognition 而非 Binding）
输入（抽象化）：
- ITC 实验对 TMPyP4 与 ZNS3 的结论为："对 GQ 序列的亲和力高于非 GQ 双链对照"。
  没有报告 Kd 或 Ka 数值。
决策：
- 若 ZNS3 是命名 G4 序列，则为 ZNS3 输出 1 条 Recognition 记录
  （activity2 = Recognition），而非 Binding 记录。
- 不得输出 value = "Higher affinity for GQ sequences (than non-GQ duplex control)"
  的 Binding 记录——这是定性比较，不是定量结合常数。
错误做法：
- 以定性比较字符串作为 value 输出 Binding 记录。
- 输出 method = ITC 且无数值 Kd/Ka 的 Binding 记录。
